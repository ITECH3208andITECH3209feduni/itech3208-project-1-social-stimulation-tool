## 🎯 Documentation Organization

### Why This Structure?
- **Clean root**: Only README.md in root for quick project overview
- **Logical grouping**: All documentation in one place
- **Easy maintenance**: Centralized documentation management
- **Version control**: Easier to track documentation changes

### Access Guidelines
- **Main README**: Use root README.md for project overview
- **Detailed docs**: Use docs/ folder for in-depth documentation
- **Links**: All docs should reference each other when relevant
- **Updates**: Keep docs in sync with code changes

### Adding New Documentation
1. Create new .md file in `docs/` folder
2. Update this README.md to include new file
3. Add relevant links from other documentation files
4. Follow established naming and formatting conventions
